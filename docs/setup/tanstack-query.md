# TanStack Query — Installation & Setup

This doc explains how `@tanstack/react-query` was added to this project, **what each step does, and why it is needed**.

---

## Background: why we need it at all

Our API SDK is generated by **Orval** (`npm run generate-sdk`). Orval was
configured to output **React Query hooks** (e.g. `useGetAllAuditLogs()`),
not plain functions. Those generated files import from `@tanstack/react-query`:

```ts
import { useQuery } from '@tanstack/react-query';
```

If the package isn't installed, TypeScript fails with:

```
Cannot find module '@tanstack/react-query' or its corresponding type declarations. ts(2307)
```

So the SDK has a hard dependency on this library. Two things are required to
make it work: **(1) install the package**, and **(2) wrap the app in a provider**.

---

## Step 1 — Install the package

```bash
npm install @tanstack/react-query
```

**Why:** This is a runtime dependency (the generated SDK calls it when the app
runs), so it goes in `dependencies`, not `devDependencies`. Installing it both
silences the TS2307 error and makes the hooks work at runtime.

Verify it landed in `package.json`:

```jsonc
"dependencies": {
  "@tanstack/react-query": "^5.x"
}
```

> If VS Code still shows the red squiggle after install, reload the TS server:
> `Ctrl/Cmd+Shift+P` → **TypeScript: Restart TS Server**. The on-disk install is
> correct; the editor just cached the old state.

---

## Step 2 — Create a QueryClient and provide it

Every React Query hook reads from a shared **cache** managed by a `QueryClient`.
That client must be made available to the whole component tree via
`QueryClientProvider`. Without it, any `useQuery` call throws:

```
No QueryClient set, use QueryClientProvider to set one
```

Edit `src/main.tsx`:

```tsx
import { StrictMode } from 'react'
import { createRoot } from 'react-dom/client'
import { QueryClient, QueryClientProvider } from '@tanstack/react-query'
import './index.css'
import App from './App.tsx'

// One QueryClient instance for the whole app. It holds the cache and
// default behaviour for every query/mutation. Created outside the render
// so it is not re-created on every re-render.
const queryClient = new QueryClient()

createRoot(document.getElementById('root')!).render(
  <StrictMode>
    <QueryClientProvider client={queryClient}>
      <App />
    </QueryClientProvider>
  </StrictMode>,
)
```

**Why each part:**

| Part | What it does | Why it matters |
|------|--------------|----------------|
| `new QueryClient()` | Creates the cache + config store | All cached server data lives here |
| Created **outside** `render()` | Single stable instance | Re-creating it per render wipes the cache and breaks caching |
| `<QueryClientProvider client={...}>` | Shares the client via React context | Hooks anywhere in the tree can find it |
| Wraps `<App />` | Covers the entire tree | Any component (now or later) can use the SDK hooks |

---

## Step 3 — Use a generated hook (example)

Once the above is in place, any component can call an SDK hook directly:

```tsx
import { useGetAllAuditLogs } from './sdk/audit-admin-controller/audit-admin-controller'

function AuditLogs() {
  const { data, isLoading, error } = useGetAllAuditLogs()

  if (isLoading) return <p>Loading…</p>
  if (error) return <p>Something went wrong</p>

  return <pre>{JSON.stringify(data, null, 2)}</pre>
}
```

**Why this is the payoff:** loading state, error state, caching, refetching, and
deduplication are all handled for you by the hook — no manual `useState` /
`useEffect` / `fetch` boilerplate.

## (Optional) Add the Devtools

A visual panel to inspect cache state, query status, and refetches during
development.

```bash
npm install -D @tanstack/react-query-devtools
```

```tsx
import { ReactQueryDevtools } from '@tanstack/react-query-devtools'

// inside the provider, after <App />
<QueryClientProvider client={queryClient}>
  <App />
  <ReactQueryDevtools initialIsOpen={false} />
</QueryClientProvider>
```

**Why optional:** it's a dev-only convenience (hence `-D`). It is excluded from
production builds and not required for the SDK to work.

---

## Summary

1. **Installed** `@tanstack/react-query` → resolves the SDK's import (TS2307).
2. **Provided** a single `QueryClient` via `QueryClientProvider` in `src/main.tsx`
   → gives every SDK hook a cache to read/write.
3. **Use** the generated hooks anywhere → free loading/error/caching handling.
